## 🗺️ 동작 방식

```
가족 A (아빠) → 거래 입력 → Apps Script → Google Sheets 저장
                                                    ↓
가족 B (엄마) ← 30초 폴링 ← "새 거래 있음" 응답 ←┘
              → 상단 배너 알림 + 피드 업데이트 → 실시간 확인!
```

**실시간 동기화 구조:**
- 모든 클라이언트가 30초마다 `poll` API를 호출
- 서버는 마지막 폴링 이후 변경된 거래와 활동 로그만 반환 (경량)
- 변경사항이 있으면 페이지를 새로고침하지 않고 DOM만 업데이트
- 다른 가족의 기록은 상단 알림 배너로 표시

## 🚀 설치 (5분)

### 1단계 — GitHub Pages 배포

```
1. 이 저장소 Fork 또는 새 저장소 생성
2. index.html, Code.gs, appsscript.json 업로드
3. Settings → Pages → Source: main / (root)
4. https://USERNAME.github.io/REPO 접속 확인
```

### 2단계 — Google Sheets 설정

```
1. sheets.new → 새 스프레드시트 생성
2. 확장 프로그램 → Apps Script
3. Code.gs 전체 붙여넣기 → 저장 (Ctrl+S)
4. 스프레드시트 새로고침
5. 💰 WEALTHOS Family 메뉴 → 🚀 시스템 초기화
6. 메뉴 → 👨‍👩‍👧 구성원 추가 (가족 이름 각각 입력)
```

### 3단계 — 웹앱 배포 (핵심)

```
Apps Script 편집기
→ 배포 → 새 배포
→ 유형: 웹 앱
→ 실행 주체: 나
→ 액세스: 모든 사용자 (익명 포함)   ← 이 설정 필수!
→ 배포 → URL 복사
```

### 4단계 — 가족 모두 연결

```
웹앱(GitHub Pages) 접속
→ 좌측 하단 ⚙️ 설정
→ Apps Script URL 붙여넣기 → 저장
→ 대시보드 자동 로드!
```

**공유 방법:** GitHub Pages URL 하나를 가족 카톡방에 공유하면 끝!
각자 같은 URL에 접속해서 본인 이름을 선택하면 됩니다.

## 📖 사용법

### 매일 쓰기

1. GitHub Pages URL 접속 (북마크 추천!)
2. 좌측에서 **내 이름 클릭**
3. 우측 상단 **➕ 거래 입력** 클릭
4. 내용, 금액 입력 → 저장

→ 30초 이내에 다른 가족 화면에 표시됩니다!

### 알림 받기

다른 가족이 거래를 입력하면:
- 상단에 배너 알림 표시 ("엄마가 외식 85,000원 기록")
- 활동 피드에 타임라인 추가
- 대시보드 집계 자동 갱신

### 목표 함께 관리

- 🎯 목표 탭에서 가족 공동 목표 추가
- 목표 카드 클릭 → 현재 금액 업데이트
- 달성률이 실시간으로 갱신되어 가족 모두에게 표시

## ❓ FAQ

**Q. 가족이 동시에 입력하면 충돌이 나나요?**
A. Google Sheets는 기본적으로 동시 편집을 지원합니다. 서로 다른 행에 추가되므로 충돌이 없습니다.

**Q. 실시간이 아니라 30초 지연이 있는데 괜찮나요?**
A. Google Apps Script의 무료 티어 제약으로 WebSocket을 지원하지 않습니다. 30초 폴링이 현실적인 최선이며, 입력 직후에는 즉시 내 화면에 반영됩니다.

**Q. 데이터가 안전한가요?**
A. 모든 데이터는 가족이 소유한 Google Drive에만 저장됩니다. GitHub Pages는 UI 코드만 호스팅하며 데이터에 접근하지 않습니다.

**Q. 구성원을 추가하려면?**
A. 스프레드시트 → `💰 WEALTHOS Family` → `👨‍👩‍👧 구성원 추가`

**Q. Apps Script 재배포 필요한 경우는?**
A. `Code.gs`를 수정했을 때만 재배포가 필요합니다. `index.html` 수정은 GitHub에 push하면 자동 반영됩니다.
